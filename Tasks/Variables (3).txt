7.1 //пять примеров правильного именования булевых переменных в вашем коде в формате "было - стало".
gameOver - isGameOver
palindrome - isPalindrome
substring - isSubstring
noMistakes - madeNoMistakes
compound - areCompound

7.2 // несколько подходящих случаев, когда в коде можно использовать типичные имена булевых переменных.
success // валидация ввода пути к файлу пользователем.
error // флаг для финализатора.
synchronized // флаг для программы, синхронизирующей две папки.

7.3
в основном даю правильные названия, в бизнес-логике делаю явные наименования, а в низкоуровневых алгоритмах для удобства испольщую однобуквенные переменные.
например такие явные наименования: 
multiplicationIteration
visitedValues
checkedElements

7.4
oldSongIndex - newSongIndex // используется для паузы текущего трека и активации нового
isPlaying - isPaused // две булевы переменные, характеризующие песню: играет она или нет. Благодаря такому подходу удобно проверять состояние песни для реадктирования модели.
worsePlayerScore - bestPlayerScore 

7.5
end - phaseEnd // в данную переменную помещается текущее время, разницу которого с временем начала игрового "тика" заношу в другую переменную и сравниваю с глобальной переменной, отражающей длительность одного "тика" в условном операторе. Данная переменная была добавлена исключительно для ясности кода, чтобы явно показать, где считывается начало "тика", а где - конец.
temporaryPlaceVariable и temporaryNameVariable // это две локальные переменные в функции для получения высшего очка из лидерборда в тетрисе. Их роль, а именно считывание побитовым сдвигом единицы и имени лучшего игрока перед тем, чтобы считать высшее очко в отдельную переменную, можно легко заменить другим способом считывания информации из файла.
